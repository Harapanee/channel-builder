# Adaptive Interview Framework

目的: ユーザーの曖昧なチャンネル構想を、channel/bible.md(12セクション)+ voice.json + .channel-system.json として確定させる。固定質問の一括表示ではなく、回答に応じて次の質問を変える。

## 進め方の原則

- ユーザーが最初の一言で語った内容から**既に確定した項目を再質問しない**(例:「歴史コメディでDoodle」と言われたらジャンル・画風は確定扱いで深掘りへ)
- 1回のメッセージで質問は2〜3個まで。選択肢を添えて答えやすくする
- 曖昧な回答は具体例で確認する(「テンポ速め、とは:1カット5秒以内/笑い20〜30秒間隔、のイメージで合っていますか?」)
- **Pilotで検証可能な不明点は仮説として先へ進む**(bible該当箇所に「仮説:」と明記)

## 確認カテゴリ → bible セクション対応

| カテゴリ | 聞くこと | 書き先 |
|---|---|---|
| コンセプト | 一言で何のチャンネルか / 視聴者への約束 / 品質の優先順位(視聴維持・理解・一貫性・事実性・独自性の順位付け) | bible §1 |
| 視聴者 | 年齢層 / 前提知識 / 視聴動機(エンタメか学習か)/ 離脱要因の仮説 | bible §2 + audience-simペルソナ |
| ナレーター | 人称(一人称/二人称/三人称)/ 人格(例: 冷静で意地悪、熱血、淡々)/ 敬体・常体 | bible §3 + 声選定の候補絞り込み |
| 台本構造 | フックの型 / 笑いと情報の比率 / 尺(**Pilot尺と本番尺を分けて聞く**。本番10分超なら章構成の希望) | bible §4 |
| 笑い | 許す笑い・許さない笑い / 頻度感 | bible §5 |
| 物語 | 定番にしたい展開の型を2〜3個(=アークメニュー) | bible §6 |
| 映像 | 画風(Doodle/漫画/モーショングラフィックス/実写素材中心…)/ 基調色 / キャラクター表現の有無 | bible §8 + style.tsパレット + コンポーネント再実装要否の判断 |
| ジャンル文法 | 題材ジャンル(歴史/科学/ビジネス…)ごとの説明の道具(地図・年表・図解・数値比較) | bible §9 |
| 素材 | AI画像の可否 / 実素材・ストックの利用方針 / 予算感 | bible §10 |
| 音 | SEの濃度(にぎやか/控えめ)/ BGMの雰囲気 | bible §11 |
| 事実性 | 史実・事実の厳密度 / 諸説の扱い / 絶対にやらないこと | bible §9・§12 |
| 運用 | 投稿頻度 / 1本あたり許容制作時間・予算 / Pilot題材 | .channel-system.json + Pilot計画 |

チャンネル名からchannelId(英小文字・ハイフンのスラッグ、例 `roman-history-comedy`)を確定したら、その直後に**フォルダ名(既定: channelId)をユーザーに確認する**(SKILL.md「ディレクトリ保護」参照。新チャンネルフォルダはファクトリールート直下に作成する)。

## 終了条件

以下がすべて満たされたらインタビュー終了:

1. bible.md の12セクションを(仮説込みで)埋められる
2. Pilot題材が1つ決まっている
3. 本番尺とPilot尺(30〜90秒)の合意がある
4. 技術前提(VOICEVOX・画像APIキー)の確認が済んでいる

## 書き出し規則

- bible.md は「教義」: 散文で書き、機械が執行できない数値の偽精度を持たせない(「約20秒に1回」は良い、「comedy: 0.45」は禁止)
- 必須の定型規則(テンプレートのbibleから継承すべきもの): 読点1文2個上限 / 間の予算 / 因果を飛ばさない / 諸説の断定禁止 / クレジット表記。これらは**画風・題材が変わっても不変のチャンネル横断規則**なので、新チャンネルのbibleにも必ず含める
- 完成した草案は全文をユーザーに提示し、承認を得てから scaffold へ進む
