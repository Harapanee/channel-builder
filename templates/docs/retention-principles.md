# リテンション(視聴維持率)原則(検証済み知見)

script-director / script-reviewer / visual-director / fact-checker / publisher が
台本・構成・ショット・約束(タイトル/サムネ)を設計・審査するときの判断基準。
チャンネル中立の原則のみを書く(フックの型・語調・章の様式はチャンネルごとの bible が
定める。本書は bible と矛盾しない範囲で適用する)。

出典: deep-research(2026-07-14、MrBeast社内文書一次PDF+本人インタビュー+YouTube公式
ヘルプ+Google RecSys 2016論文のライブ取得、全クレーム3票の敵対検証)。要約と出典一覧は
ファクトリールートの docs/retention-principles-research.md。

## 検証済み原則(設計に使う)

1. **企画評価は CTR × 期待視聴時間の積**(確度: 高)
   YouTube推薦の目的関数は「インプレッションあたりの期待視聴時間」(Google自身の論文)。
   CTRだけ高い釣り題材も、視聴時間だけ長い地味題材も伸びない。題材・タイトル・構成は
   「クリックしたくなり、かつ目標尺ぶん見たくなるか」で評価する。

2. **二段フック: 0〜30秒=約束の回収、30〜60秒=期待超え**(確度: 高)
   ほぼ全動画で離脱が最大なのは最初の1分。YouTube公式「イントロ指標」は30秒地点の
   継続率を「内容が期待と一致したか」の判定に使う。0〜30秒でタイトル・サムネの約束に
   正面から応え(前置き・自己紹介・経緯説明を置かない)、30〜60秒で期待を超える要素
   (約束の先にある意外性)を1つ見せる。

3. **約束が先、台本が回収する**(確度: 高)
   タイトル・サムネは期待設定装置であり、期待一致に20秒かかると視聴者を失う。
   約束(仮タイトル+サムネ一言の方向)は調査段階で確定し(research.md「約束」節)、
   台本冒頭が回収し、公開パッケージはその最終化として作る。台本より後に約束を発明しない。

4. **再フックの計画配置**(確度: 高。位置の最適値は中)
   長尺は単一のストーリーラインだけでは注意を維持できない。3分・6分地点(目安。
   尺比率なら約22%・44%地点)に「このチャンネルでしか聞けない」情報・転回を計画的に
   配置する。絶対時間か尺比率かは未検証 — 自チャンネルの実測カーブで検証する。

5. **フロントロード: 説明は後半へ**(確度: 高)
   前半(〜6分)は最も刺激的で単純な内容+速い画面転換。前提説明・詳細解説・背景知識は
   6分以降(視聴者が没入した区間)へ置く。YouTube公式推奨「魅力的なコンテンツの前方
   移動」と独立に相互裏付けされている。

6. **ペイオフは最後まで伏せ、終わりは唐突に**(確度: 高)
   冒頭で開いた問いの答え・結末は終盤まで明かさない。規模・意外性は段階的にエスカレート
   させる(stair stepping)。終端は「回収 →(規約上の義務表記のみ)→ 即終了」。
   エンドトーク・次回予告・チャンネル紹介・余韻を置かない。終わりを匂わせる合図は
   ペイオフの煽り以外禁止。

7. **画面変化のリズム**(確度: 中)
   同一画面が10秒以上続くと離脱要因になる。カットまたは画面内の視覚変化を10秒以内に
   必ず入れる(bible の画面リズム規定がより厳しい場合はそちらに従う)。

## 適用上の留意(必ず読む)

- 全知見は英語圏・出演者ありのエンタメ動画由来。日本語スライドショー型・ナレーション型
  への適用効果は**未実証** — 上の原則は構造原則からの推論であり、**実測(イントロ指標・
  維持率カーブ)が最終判断者**。原則と実測が食い違ったら実測に従い、/channel-refine で
  原則の適用方法を直す。
- 「リテンション70%」はMrBeast級の参照値。合否・目標に使わない。目標は自チャンネル
  実測の改善傾向とする。
- 引き延ばしによるAVD稼ぎは禁物。「前回の動画の体験」が次の視聴確率を決める(データに
  出ない連鎖効果)上、YouTubeは満足度サーベイで引き延ばしを検出する方向に進化している。
- 「ここでしかできない再エンゲージメント」は、非顔出しチャンネルでは「このチャンネルで
  しか聞けない情報・展開・切り口」へ翻訳する。

## 機械検査との分担

- 約束の冒頭回収・再フック配置・終端の作りは script-reviewer(合否権)と
  compliance-reviewer(review-checklist)が検査する(機械lintの対象外)。
- ショットの画面変化リズム・role配置は visual-director が自己計測し、qa-smoke が
  静止フレームを機械検出する。

## 運用ループ(実測が最終判断者)

1. 公開後、factory-ui の「分析取得」で維持率カーブを取得する(イントロ指標・最大の離脱・
   最大の山が要約表示される)
2. /channel-refine が要約と bible の該当節を突合し、脱落パターンをチャンネル教義へ還元する
   (冒頭30秒の脱落→フック規則、中盤の段差→章構成・再フック位置、終端の急落→終わり方)
3. 数値規則(再フックの最適位置・冒頭の許容前置き秒数など)は自チャンネル実測の蓄積から
   導出する(目安: 同一チャンネルで5本以上)
