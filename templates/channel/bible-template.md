**固定アウトロ(不変規則)**: 全動画の末尾に毎回同じ約6〜8秒のアウトロ(チャンネル名カード+固定ナレーション+クレジット、共有カスタムOutro)を置く。文言はチャンネルごとにここで定義する。

     ヒアリング「台本構造」カテゴリ → 本セクション。時間は目安であり機械検証しない。 -->

<目標尺の基本骨格を時間帯で割り付ける。例: 0〜10秒フック / 10〜25秒設定 / … / 終盤オチと回収。>

**【不変規則】構造規則**:

- 冒頭で「この動画は何の話で、何が焦点なのか」が早期に分かること
- 説明が続いたら感情か笑い(またはこのチャンネルの興味の核)を挟むこと
- 終盤は必ず冒頭の何かを回収すること(台詞・構図・SE、どれでもよい)
- 「続きが気になる問い」を最低1つ、冒頭で開いて終盤まで引っ張ること
- **同型ビートは3連続まで**: 同じ形の出来事を4つ以上並べない。4つ目の前に視点転換か異質なビートを挟む(長尺で特に検査)
- **誤読の検査**: 音訓の取り違えが起きやすい語(「者」「実の」等)はひらがなに開くか言い換える。TTS後に narration/readings.md を reading-checker で必ず検査する(不変規則)。実測で頻出の誤読: 氏族名・家名の「〜家」(→カ。正: け)、音訓交ぜ語=湯桶・重箱読み(例: 「格上」→カクジョウ。正: かくうえ)。読みはひらがな表記で固定し、字幕の見た目は display注釈(行直下の `- display: <漢字表記>`)で保てる
- **因果を飛ばさない**: 「結果」「そのせいで」「極めつけ」などの接続語は、直前の行に指示対象(何の結果か)が明示されているときだけ使う。台本完成後、行を順に読み「この行は前の行から論理的につながるか」を全行で自問する。飛躍が見つかったら接続の一句を足すか、行を分ける。短い尺では圧縮によって因果が飛びやすいので特に注意する
- **1情報1行(冗長禁止)**: 直前の行と同じ情報の言い換え・念押し・補強だけの行を書かない。同一情報の再言明が許されるのは (a)章payoffでの意味の転換(新しい視点が加わるとき) (b)冒頭モチーフの終盤回収、の2つだけ。台本完成後「この行を削ると意味が欠けるか」を全行で自問し、欠けない行は削る(補強行が積もった台本は視聴者に「話が進んでいない」と感じさせ、中だるみ離脱の主因になる)
- **テンポの緩急**: 同じ長さ・同じdelivery・同じポーズ幅の行を10行以上連続させない。各章に最低1箇所、短文の連打か長い間(音の設計の予算内)で拍を変える

> 例(「転生したら〜」60〜90秒): 0〜10秒=転生の宣告と最悪条件の提示 / 10〜25秒=状況設定 / 25〜45秒=「救い」を1つずつ潰す / 45〜70秒=どん底または転回 / 70〜90秒=冒頭モチーフを意味を変えて回収。

## 7. 視聴者体験設計の手引き

<!-- 何を書くか: ストーリーボード作成時の思考ツール。この節はチャンネルを問わずほぼ共通なので、
     下記をそのまま残してよい(必要なら例だけ差し替える)。JSONにせず storyboard.md に散文で書く。 -->

- **中心の問い**: 動画全体を牽引する1つの疑問。冒頭で開き、終盤で回収する。部分回答を中盤に1〜2回置く
- **視聴者状態の入口と出口**: 各シーンの前後で「知っていること/思い込んでいること/感じていること/次に期待すること」がどう変わるか。変わらないシーンは削るか統合する
- **Reveal / Withhold**: 全情報を即時に見せない。決定的情報は、見せる瞬間の演出(カメラを引く、数える、増殖させる)のために取っておく
- **ショットの役割**: show / explain / contrast / foreshadow / withhold / reveal / payoff / gag / reframe。全ショットが show と explain だけの動画は退屈な解説動画である
- **パターンと破壊**: 意図的に反復を確立してから崩す
- **モチーフの再登場**: 同じ形・色・音を意味を変えて再登場させる
- **ピーク予算**: 最大の色数・音量・速度・新規性はクライマックスのために予約する。序盤で全力を出さない
- **音との同期**: lead(映像が先行)/ confirm(同時)/ lag(音が先)。ショットごとにどれかを意識する

> 例(「転生したら〜」): オフホワイト背景/ラフな黒マーカー線/藍+赤+黄の最小色/頭身1/3・点目・陰影なし。マッピング=危険→赤い手描き円 / 力の差→主人公を極端に小さく描く / 失敗→キャラを押し潰す / 希望→黄色の光 / 死→チーン(SE)+白目+水平倒れ(リアルな死描写は使わない)。

**図解部品は実差があるときだけ**(全チャンネル共通の不変規則): 大小比較(ComparisonSplit等)は数量・規模に実差(目安1.5倍以上)がある事実にのみ使う。抽象概念の対比を差のない図形+文字ラベルで済ませない — 概念の対比は絵(素材・カスタム演出)で見せる。

## 9. ジャンル文法

<!-- 何を書くか: 題材ジャンル(歴史/科学/ビジネス/料理…)ごとの説明の道具(タイムライン・地図・図解・
     数値比較・出典 等)/ このジャンルで避けるべき定型表現。ヒアリング「ジャンル文法」カテゴリ → 本セクション。 -->

説明の道具は次を優先する: <このジャンルで理解を助ける表現手段を列挙(例: 時間/地理/勢力/因果/出典)>

- 図解・数値・年表など「事実を扱う画面」では誇張しない。誇張(ギャグ)と事実(説明)を画面上で混同させない
- 避けるべき定型: <このジャンルで陳腐化した表現を列挙>
- **【不変規則】諸説の断定禁止**: 諸説ある事柄は「〜と言われています」で逃げず、最有力説を採るか、諸説あること自体をネタにする。断定できない事実を断定しない
- **【不変規則】地理形状は実データ**: 大陸・海岸線・国土の輪郭をフリーハンドで発明しない。実データからトレースした共有ジオメトリ(japan-geometry / world-geometry)だけを使う。無い地域が必要になったら不足部品として報告し、実データトレースで shared に新設してから使う
- **【不変規則】地図は主張があるときだけ**: 地図ショットは台本の行の地理的主張(位置・経路・距離・広さのどれか)をちょうど1つ持ち、それをprops(ハイライト・ルート・ズーム)で表現する。表現できないなら部品を拡張するか別の演出にする。「とりあえず地図を映す」は禁止

> 例(「転生したら〜」歴史): 道具=時間(タイムライン)/地理(地図)/勢力(色分け・大小)/因果(矢印)/史料(出典)。避ける定型=肖像画+年号字幕の反復、ナレーションと無関係な合戦イラストの垂れ流し。

## 10. 素材戦略

<!-- 何を書くか: AI画像の可否 / 実素材・ストックの利用方針 / 予算感 / キャラクター表現の有無。
     ヒアリング「素材」カテゴリ → 本セクション。取得優先順位と【不変のキュレーション規則】は残す。 -->

ショットごとに上から順に検討する:

1. **Channel Asset Library**(既存素材の再利用 — library.json 登録済みのもの)
2. **コード生成**(SVG / Remotionコンポーネント)— 図形・数値・比較・小物・地図
3. **パブリックドメイン/許諾済み素材**
4. **AI画像**(参照画像方式)— キャラクター・複雑な小物
5. AI動画は使用しない

**【不変規則】キャラクター素材と人間キュレーション**:

- 人物ごとに正典(canonical)1枚を人間が承認し、全バリアントは正典を参照画像にして生成する
- バリアントは表情4種+ポーズ系4種の計8枚以内。それ以上の表現差は motion ヘルパー(揺れ・潰れ・跳ね)で作る
- 透過前提のAI素材は必ず純緑のグリーンバックで生成する(§8のプロンプトに内蔵)。除去は remove-bg.ts が自動判別(緑→クロマキー+スピル補正 / オフホワイト→flood-fill)
- **全素材は人間キュレーション(採用/却下)を経て library.json に登録(approvedBy: "human")されてから使用可能**。未登録素材はショットから参照できない
- 主要な敵役・対役も文字だけで済ませず素材化する。クライマックスの勝敗・転回・決定的瞬間は絵で見せる

> 例(「転生したら〜」): 場所・場面(城・街並み・合戦場)も SVG で伝わらない場合は AI 画像で素材化してよい(low detail、キャラより目立たせない)。

## 11. 音の設計

<!-- 何を書くか: SEの濃度(にぎやか/控えめ)/ BGMの雰囲気 / 間の予算。
     ヒアリング「音」カテゴリ → 本セクション。テンプレート同梱の SEパック24種+BGM2曲が初期在庫
     (assets/audio/、library.json 登録済み)。合わない場合のみ差し替える。 -->

- **SEの役割**: <このチャンネルでSEが担う仕事(句読点・強調・場面転換など)を書く>
- **無音の使い方**: 大きな山の直前に短い無音を作ってから鳴らす。常時鳴らさない
- **BGM**: 基本1曲を小さく敷く。BGMはチャンネルの「音の署名」なので題材ごとに変えず固定するのが既定。クライマックスで止める/変えるのは効果的だが毎回はやらない
- **【不変規則】間の予算**: ナレーションの間は script.md の pause_after_sec で制御する。長い間(0.6秒超)は「ここぞ」に限り、1本あたり5箇所以内を目安にする。通常の行間は0.3〜0.4秒、オチ・ツッコミ前は0.5〜0.7秒
- **【不変規則】SEの定量予算**(shots.json設計時に自己計測): SE総数は**尺(秒)÷8以下**(平均8秒に1回以下)。全SEは機能(強調・衝撃・転換・回収など)を一言で言えるものだけ。**同一SEは総数の20%まで**。画面が変わっただけ・文字が出ただけでは鳴らさない
- **【不変規則】音量**: ナレーションが常に主役。SE・BGMがナレーションを聞き取りにくくしたら即失格(Mechanical QA のラウドネス基準は -14 LUFS)

> 例(「転生したら〜」): ボケ直後にズコー、衝撃にドン、死にチーン。和風ワンショット(太鼓・拍子木)を場面転換の記号として一貫使用。

## 13. 公開パッケージ(タイトル・サムネイル・概要欄)

動画1本ごとに、タイトル3案・サムネイル3案・概要欄1案を生成する(publisherエージェントの責務)。

**タイトル(3案、戦略別に必ず1つずつ)**:

- A 誤解破壊型: 常識を裏切る事実を前に出す(「実は〜だった」)
- B 数字インパクト型: 効かせる数字を前に出す
- C 感情・自分事型: チャンネルの署名タイトルの型を活かす
- 共通規則: 28文字以内(検索結果での切れ防止)/ 動画が実際に答える範囲を超える釣りは禁止 / 記号の乱用禁止

**サムネイル(3案)**: <方式を選んで固定: 「絵はAI生成(asset-generator型5)+文字合成はRemotion」または「既存素材+Remotion部品構成」>。**3案は人間が1つを選ぶのではなく、YouTube Studioの「テストと比較」へそのまま3枚投入してABテストする**(実データが選ぶ)。

- **構造はチャンネルで固定**: <例(AI生成方式): その回の「最悪の瞬間」を描くフルフレーム1枚絵(本編と同じ画風・主人公は正典 `--ref` 参照・表情/状況を強く誇張)+ 事実型の一言(8文字以内・感情中立)>
- 3案は「<場面/表情> × 一言の切り口」の違いで作る(切り口の例: 状況 / 数字 / 皮肉)
- 注目オブジェクトは少数に。一言の予定領域は絵をシンプルに(docs/thumbnail-principles.md の検証済み原則に従う)。補助アクセントは最大1個
- 共通規則: 一言は太い縁取りでスマホでも読める大きさ(文字色はパレット名のみ)/ 動画内容と無関係な絵の禁止 / AI生成画像を使う場合はStudio投入前に人間が確認する

**概要欄(1案)**: 次の順で構成する。

1. 要約1〜2文(中心の問いを、答えをバラさずに)
2. 補足(諸説がある題材はその旨を明記)
3. 主要な出典3〜5件(research.mdから)
